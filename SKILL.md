# Data QA Skill

本 Skill 帮助用户对 TPO 数据文件（xlsx/csv）进行智能问答分析。**核心特点**：
- **深度自然语言理解**：支持复杂的问答场景，智能识别用户意图
- **多维度分析能力**：环比、同比、趋势、排名、对比、异常检测
- **答案自动验证**：自动检查答案的准确性和完整性
- **单月分析模式**：默认返回最新月数据，支持指定月份查询
- **场景数据按实有**：有啥说啥，不编造缺失场景

## 业务领域知识（必读）

以下是该数据集的核心业务定义，问答时必须遵守：

### 核心指标定义

| 指标 | 定义 | 计算公式 |
|------|------|---------|
| **TPO** | 人工进线量与完单量的比值，衡量人工效率 | `人工进线量 / 完单量 × 10000` |
| **YOY** | 年度同比（Year over Year），对比上一年同月数据 | `(本期值 - 上年同期值) / 上年同期值 × 100%` |
| **MoM** | 环比（Month over Month），对比上个月数据 | `(本期值 - 上月值) / 上月值 × 100%` |

### 场景名称映射

| 原始字段名 / 旧名称 | 当前业务名称 | 说明 |
|---------------------|-------------|------|
| 微信渠道客诉 | **资产权益** | 渠道已更名，两个名称均指同一渠道 |
| 二次号 | **售后服务** | 渠道已更名，两个名称均指同一渠道 |

### 十大场景数据规则

**核心原则：有啥说啥，不编造缺失场景**

数据文件中实际包含的场景会因月份而异：
- 2025/1-2025/10：费用疑义、营销活动、二次号（3个场景）
- 2025/11-2026/3：费用疑义、营销活动、售后服务、资产权益（4个场景）

**问答规则**：
- 如果某月份只有 3 个场景，就只展示这 3 个，**不要说"缺失 7 个"**
- 如果某月份有 4 个场景，就展示这 4 个
- **禁止编造不存在的场景**（如"费用疑义、营销活动、售后服务、资产权益、..."列出10个）

### 空值处理规则

- **2025/11 和 2025/12 的 YOY 字段为空值**：这是正常现象，因为没有对应的往年数据，**不需要提示异常，不需要处理，直接忽略**。
- 其他月份的空值 YOY 如出现，才需要注意并报告。

## 工作流程

用户提供数据文件时，按以下步骤处理：

### Step 1：文件接收与记忆检查

```
1. 用户上传 xlsx/csv 文件（或提供路径）
2. 运行 scripts/data_parser.py 解析文件结构
3. 检查 memory/datasets.json 是否已有该文件的记忆（基于文件哈希）
   - 已有记忆 → 加载记忆摘要，告知用户"已从记忆中加载数据结构"
   - 无记忆 → 执行完整解析，存储数据记忆
```

### Step 2：数据结构理解

```
1. 识别所有列名（字段名）及其数据类型
2. 检测时间序列列（月份、日期等）
3. 识别指标列（数值型，如 TPO、YOY、进线量、完单量）
4. 识别维度列（分类型，如渠道名称、产品线）
5. 应用业务语义映射（见上方业务领域知识）
```

### Step 3：问答处理

收到用户问题后：

```
1. 运行 scripts/qa_engine_v4.py 进行智能问答
   - 意图识别：自动识别用户问题的意图类型（查询、排名、趋势、对比、异常检测）
   - 数据分析：根据意图执行相应的数据分析操作
   - 答案生成：生成结构化的答案（包含结论、详细数据、趋势对比）
   - 答案验证：自动检查答案的准确性和完整性
2. 格式化输出结果
   - 使用统一的输出格式：问题、月份、结论、详细数据、趋势对比
   - 支持多种意图类型的差异化输出
3. （可选）将新 Q&A 对存储到 memory/qa_memory.db
```

### Step 4：结果输出格式（严格规范）

**核心原则**：
1. **不累计**：不计算多个月的累计变化（如"累计下降 X%"），只展示当期数值
2. **精确描述**：禁止使用模糊形容词（如"剧烈"、"明显"、"显著"），所有趋势判断必须有数据支撑
3. **默认最新月**：用户未指定月份时，默认返回最新月份的数据
4. **仅相邻对比**：趋势仅与上个月和去年同期对比，不与任意历史月对比
5. **场景按实有**：有啥说啥，不编造缺失场景

输出结构：
```
## 📊 查询结果

**问题**：[用户原始问题]

**月份**：[数据所属月份，默认最新月]

**结论**：[一句话核心答案，必须包含数值]

**详细数据**：
[表格或列表]

**趋势对比**：
- 环比（vs上月）：[有数据则写，无数据则写"-"]
- 同比（vs上年同月）：[有数据则写，无数据则写"-"]
```

## 脚本调用方式

### 智能问答（推荐）

```bash
python scripts/qa_engine_v4.py --file <csv_path> --question "<问题>"
```

支持的问答类型：
- **月份数据查询**："最新月的费用疑义场景数据"、"2026/3 的 TPO 数据"
- **场景查询**："各场景的 TPO"、"费用疑义的占比"
- **趋势查询**："最新月的趋势"、"同比和环比情况"
- **排名查询**："排名前3的场景"、"TPO 最高的场景"
- **异常检测**："检测异常"、"查看是否有波动"
- **对比查询**："对比最近两个月"、"2026/2 vs 2026/3"

输出格式：
```
## 查询结果

**问题**：[用户原始问题]

**月份**：[数据所属月份]

**结论**：[一句话核心答案，包含数值]

**详细数据**：
[表格或列表]

**状态**：[OK]/[!]/[X]
```

### 单月分析（简单模式）

```bash
python scripts/qa_engine_v3.py --file <csv_path> [--month <YYYY/M>]
```

输出：
- 单月 TPO 分析报告（完单量、总 TPO、年度 YoY、相邻月趋势、场景 TPO）
- 默认分析最新月（文件最后一行）
- 指定 `--month` 参数可查询特定月份

### 解析文件

```bash
python scripts/data_parser.py --file <xlsx_or_csv_path> --memory-dir memory/
```

输出：
- 字段摘要（字段名、类型、示例值、统计信息）
- 文件哈希（用于记忆检查）
- 是否从记忆加载（`from_cache: true/false`）

### 记忆管理

```bash
# 查看所有数据记忆
python scripts/memory_manager.py --list-datasets

# 查看问答历史（最近 20 条）
python scripts/memory_manager.py --list-qa --limit 20

# 清除指定文件的记忆
python scripts/memory_manager.py --clear-dataset <file_hash>

# 清除所有问答记忆
python scripts/memory_manager.py --clear-qa-all

# 搜索历史问答
python scripts/memory_manager.py --search-qa "<关键词>"
```

## 常见问题类型与处理策略

### 快速参考

| 问题类型 | 示例 | 处理策略 |
|---------|------|---------|
| 月份数据查询 | "最新月的费用疑义场景数据"、"2026/3 的数据" | 返回指定月份的完整分析报告 |
| 场景查询 | "各场景的 TPO"、"费用疑义的占比" | 返回场景数据，包含 TPO、YoY、占比 |
| 趋势查询 | "最新月的趋势"、"同比和环比情况" | 返回总TPO的同比和环比趋势 |
| 排名查询 | "排名前3的场景"、"TPO 最高的场景" | 按TPO排序，返回Top N场景 |
| 异常检测 | "检测异常"、"查看是否有波动" | 检测YoY或MoM变化超过阈值的场景 |
| 对比查询 | "对比最近两个月"、"2026/2 vs 2026/3" | 对比两个月份的场景数据变化 |

**v4.0 新增能力**：
- ✅ 智能意图识别：自动理解用户问题类型
- ✅ 多维度分析：支持6种查询类型
- ✅ 答案验证：自动检查答案准确性
- ✅ 异常检测：可配置阈值检测数据异常
- ✅ 场景对比：对比不同月份的场景变化

**禁止的计算类型**：
- 累计计算（如"累计增长 X%"、"累计下降 Y%"）
- 多月聚合（如"2025 年上半年平均"）
- 非相邻趋势对比（如"vs 2025/1"、"vs 历史最高"）
- 编造缺失场景（如数据只有 4 个场景，却说"十大场景 TPO 分别是..."）

## 注意事项

1. **所有数据存储在本地**，不上传云端，保护数据隐私。
2. **渠道名称**：回答时统一使用**当前业务名称**（资产权益、售后服务），不使用旧名称。
3. **YOY 空值**：2025/11 和 2025/12 的 YOY 为空属于正常，回答中无需提及此异常。
4. **场景数据**：按实有场景展示，有 3 个就展示 3 个，有 4 个就展示 4 个，**禁止编造其他场景**。
5. **数值精度**：默认保留2位小数；用户指定精度时全局应用。
6. **默认最新月**：用户未指定月份时，默认分析最新月（文件最后一行）。
7. **趋势对比**：仅与相邻月和去年同期对比，不统计多月趋势。
