# Data Structures — 数据结构识别指南

本文档定义数据集中各类字段的识别规则与业务含义，供 AI 在解析文件时参考。

---

## 字段类型自动识别

| pandas 类型 | 语义类型 | 识别规则 |
|------------|---------|---------|
| int64 / float64 | **numeric**（数值型） | pandas dtype 为数值 |
| object（内容为数字） | **numeric**（数值型） | 超过 80% 的非空值可转为 float |
| datetime64 | **datetime**（时间型） | pandas dtype 为 datetime |
| object（可解析为日期） | **datetime_string**（时间字符串） | 前5个非空值超过3个可转 datetime |
| object（唯一值少） | **categorical**（分类型） | 唯一值占比 < 30% 且数量 < 50 |
| object（其他） | **text**（文本型） | 其余情况 |

| 字段关键词 | 语义类型 | 单位 | 说明 |

## 特殊字段规则

### TPO 字段

- **直接字段**：列名直接包含"tpo"，直接读取
- **动态计算**：数据中无 TPO 列时，用以下公式计算：
  ```
  TPO = 人工进线量 / 完单量 × 10000
  ```
- **精度**：保留 2 位小数

### YOY 字段

- **数值格式**：通常为小数（0.1234 = 12.34%），部分数据集可能直接为百分比形式（12.34）
- **识别方式**：若 YOY 列均值在 -1 到 1 之间，视为小数格式；若均值在 -100 到 100 之间，视为百分比格式
- **空值豁免**：2025/11 和 2025/12 的 YOY 字段为空值属于正常，**不需要任何处理**

## 数据完整性检查

解析文件后，自动检查以下项目：

| 检查项 | 通过条件 | 失败处理 |
|--------|---------|---------|
| 时间列存在 | 存在至少一个可识别的时间列 | 提示"未找到时间列，趋势分析不可用" |
| 数值列存在 | 存在至少一个数值列 | 提示"未找到数值列，无法进行指标分析" |
| 数据行数 | 行数 > 0 | 提示"文件为空或无有效数据" |
| 渠道列存在 | 存在维度列 | 提示"未找到渠道/维度列，无法按渠道分析" |
| YOY 空值比例 | 豁免月份外的 YOY 空值 < 50% | 提示"YOY 数据缺失较多，结果仅供参考" |
